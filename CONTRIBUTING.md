# Contributing

Thank you for your interest in contributing to country_code_picker!

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/lukman48/country_code_picker.git`
3. Create a branch: `git checkout -b feature/your-feature`
4. Make your changes
5. Test thoroughly: `flutter test`
6. Commit: `git commit -am 'Add your feature'`
7. Push: `git push origin feature/your-feature`
8. Submit a Pull Request

## Code Style

- Follow Dart style guide: https://dart.dev/guides/language/effective-dart/style
- Use `dart format` to format code
- Run `dart analyze` and fix all warnings

## Testing

All new features must include tests:

```bash
flutter test
```

Maintain 90%+ code coverage.

## Commit Messages

- Use clear, descriptive messages
- Start with a verb: "Add", "Fix", "Update", "Remove"
- Keep first line under 72 characters
- Reference issues: "Fixes #123"

## Pull Request Process

1. Update README.md if needed
2. Update CHANGELOG.md
3. Ensure all tests pass
4. Get approval from maintainer
5. Merge and celebrate! 🎉

## Areas for Contribution

- 🌍 Add more countries/territories
- 🎨 Improve UI/UX design
- 📱 Mobile-specific improvements
- ♿ Accessibility enhancements
- 📚 Documentation improvements
- 🧪 More test coverage
- 🐛 Bug fixes
- 🚀 Performance improvements

## Reporting Issues

- Check existing issues first
- Provide minimal reproducible example
- Include Flutter version: `flutter --version`
- Include package version
- Describe expected vs actual behavior

## Code of Conduct

Be respectful, inclusive, and professional. We're here to help each other!
